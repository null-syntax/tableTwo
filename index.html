tableTwo

TableTwo is a Bootstrap compatible jQuery plugin that easily turns HTML tables into a responsive editable environment.

Getting Started

For tableTwo elements to render the following two files must be included from repository above:

      tableTwo.js
      tableTwo.css

Tables can then be initiated using jQuery selectors as shown below:

      
      $('#elementToTableify').tableTwo();
or: 
      
      $('.elementsToTableify').tableTwo();

Due to the level of customisation provided by tableTwo it is likely you will need to override atleast some of the default options.

This can be achived by passing an object to the element when it is initialised, for example:

      
      $('.elementsToTableify').tableTwo({
      addRow:false,
      showRowId:true,
      });

Receiving Data

tableTwo will work with both static and JSON data, by default tableTwo will render a static table into a tableTwo element.

JSON Data

When receiving json data the 'getJsonData' flag must be enabled and the 'jsonurl' value set. You can also specify the POST data for the request by passing values to the 'getJsonPOSTData' object: 


$('#new-table').tableTwo({
  getJsonData:true,
  jsonurl: "http://ilove.json.com"
  getJsonPOSTData{'request': id, 'user': id}
  });

When using JSON data tableTwo will render best receiving an array of objects, a new column header will be generated for each unique key found when iterating through objects in the array.

Callbacks

Callbacks are initiated after manipulation of the element has been completed but before any calls to send functions are completed.

For callbacks involing manipulation of a cell, row or column the 'this' object will contain access to the data referenced in the cells affected by the calling function.

Currently the sendOnChange function uses AJAX to send a full representation of the table to the url passed to 'sendJsonUrl'.

Creating callbacks

To create you own custom callback function simple pass you own function to tableTwo when initialising your element: 


$('#new-table').tableTwo({
  removeRow:false,
  showRowId:true,
  onRowAdd: function(){
    alert('You added a row containing ' + JSON.stringify(this) + '!');
  },
});

Adding Column Types

Custom column types can also be added allowing you to easily add your own custom toggles, select menus and more.

First ensure the flag 'enableCustomColumnTypes' is set to true. Then pass an object to 'columnTypes' containing values in the following format:

Basic Example


$('#new-table').tableTwo({
  columnTypes:{
    //example of a custom 'text' column
      text:{            // This is the name of your column type -- UNIQUE
        id:9,           // must start at 9 or above and be -- UNIQUE
        title: "text",  // Value displayed to user in menu systems
        type: "text",   // if overloading existing column type reference it here
        value: ""       // default value
      }
    }
  });

Date Column


  $('#new-table').tableTwo({
      columnTypes:{
      //example of a custom date column
      date:{            // This is the name of your column type -- UNIQUE
        id:10,          // must start at 9 or above and be -- UNIQUE
        title: "date",  // Value displayed to user in menu systems
        type: "date",   // reference 'date' column type
        value:"now"     // default value
      },
    }
  });

Select Column

  
    $('#new-table').tableTwo({
      columnTypes:{
    //example of a custom select column
     select:{                 // This is the name of your column type -- UNIQUE
       id:11,                 // must start at 9 or above and be -- UNIQUE
       title:"select",        // Value displayed to user in menu systems
       type: "select",        // reference 'select' column type
       values: ['']           // default values (array)
     },
    // custom 'select' example 1
    status:{
      id:12,
      title:"status",
      type:"select",
      values: ['created','started','done']
    },
    // custom 'select' example 2
    contactTypes:{
      id:13,
      title:"Contact Types",
      type:"select",
      values: ['Mobile','Email','SMS']
    },
    }
  })

Toggle Column

When using toggle columns it is easy to customise the appearance of you buttons by passing values to the columnTypes object.

When initialising your element you can preset the value of a toggle cell by passing "true" or "false". This works for both static and JSON data.

By default toggle columns work using font awesome icons however this can be overwritten by any similar class based icon system. 

  
    $('#new-table').tableTwo({
      columnTypes:{
        //example of a custom select column
        star:{                    // This is the name of your column type -- UNIQUE
          id:14,                  // must start at 9 or above and be -- UNIQUE
          title:"star",           // Value displayed to user in menu systems
          type:"toggle",          // reference 'toggle' column type
          on:"fa fa-star-o",      // font awesome icon class
          off:"fa fa-star"        // font awesome icon class
        },
        check:{
          id:15,
          title:"done",
          type:"toggle",
          on: "fa fa-check",
          off: "fa fa-times"
        },
        switch:{
          id:16,
          title:"switch",
          type:"toggle",
          on: "fa fa-toggle-on",
          off: "fa fa-toggle-off"
        }
      }
    });

User Menus and UI elements

tableTwo contains a number of user menus and UI elements which can be enabled or disabled as required.

Column Options Menu

The column options menu can be used to dynamically change the type of columns in a table.

It can also be used to change the table header or amend the layout of a column and is opened by clicking the icon in the column header.

This menu can be disabled by setting the 'showColumnOptions' flag to false when initialising the element:

When enabled this option overrides the 'editHeaders' option / bindings but provides the same functionality. 



  $('#new-table').tableTwo({
    showColumnOptions:false,
  });

Footer Menu

The footer menu can be customised with a number of options adding additional functionality to your table.

It can replace or augment the 'addRow' functionality with a bootstrap popover, and also houses the submit button if this option is enabled. 


  $('#new-table').tableTwo({
    addFooter:true,
    footerAddRowButton:true,
    footerShowSubmitButton:true,
  });
